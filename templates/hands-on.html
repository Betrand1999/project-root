Practical Exercises and Scenarios

1. Deploy a Simple Web Application on AWS

Task: Create an EC2 instance, install a web server, and host a simple web page.

  • Launch an EC2 instance with a public IP.
  • Install Apache or Nginx web server.
  • Configure security groups to allow HTTP traffic.
  • Deploy a static HTML page and verify accessibility over the internet.

2. Build a CI/CD Pipeline with GitHub Actions

Task: Automate the deployment of a codebase using GitHub Actions.

  • Set up a repository with sample code.
  • Create a GitHub Actions workflow for build, test, and deployment stages.
  • Integrate testing tools and notify on build status.

3. Infrastructure as Code with Terraform

Task: Provision and manage cloud resources using Terraform.

  • Write a Terraform configuration to create an S3 bucket.
  • Apply changes and validate resource creation.
  • Update the configuration and test for idempotency.

4. Kubernetes Deployment with Helm

Task: Deploy and manage a Kubernetes application using Helm charts.

  • Set up a Kubernetes cluster (use Minikube or a cloud provider).
  • Install Helm and create a Helm chart for a sample application.
  • Deploy the application and manage versioning with Helm upgrades.

5. Configure Monitoring with Prometheus and Grafana

Task: Set up a monitoring stack for a web application.

  • Install Prometheus and Grafana on a server or container environment.
  • Configure Prometheus to scrape metrics from your web application.
  • Create custom dashboards in Grafana for visualizing application metrics.

6. Create a Secure Environment with Ansible

Task: Use Ansible to automate security configurations on servers.

  • Write an Ansible playbook to configure firewall rules.
  • Ensure SSH key-based authentication is enforced.
  • Apply security patches and audit server configurations.

7. Dockerize a Python Application

Task: Containerize a simple Python Flask application using Docker.

  • Create a `Dockerfile` for the Flask application.
  • Build and run the Docker image locally.
  • Test the application by accessing it through the container port.
